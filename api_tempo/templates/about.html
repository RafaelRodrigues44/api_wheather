Esta API, construída em Django com Django Rest Framework (DRF), é uma RESTful API que segue o padrão MVT (Model-View-Template).

O padrão MVT é uma variação do padrão MVC (Model-View-Controller), comumente usado em frameworks web como o Django. Aqui está uma breve explicação de cada componente:

  • Modelo (Model): Representa a estrutura de dados da aplicação. No caso desta API, os modelos definem a estrutura das previsões do tempo e dos usuários.
  • Visão (View): Lida com a lógica de apresentação e interação do usuário. As views são responsáveis por processar solicitações HTTP e retornar respostas apropriadas. As views neste contexto são implementadas como classes de visualização do Django.
  • Template (Template): Fornece a estrutura de apresentação para as respostas das views. Os templates são arquivos HTML que podem incluir dados dinâmicos renderizados pelas views.

Além disso, esta API utiliza o banco de dados NoSQL MongoDB para armazenar os dados das previsões do tempo. O MongoDB é uma escolha comum para aplicações que exigem escalabilidade e flexibilidade no esquema de dados.

Endpoints Disponíveis

  • Usuários:
    • GET /api_tempo/users/: Retorna a lista de usuários.
    • POST /api_tempo/user/create/: Cria um novo usuário.
    • GET /api_tempo/user/get/:email/: Retorna os detalhes de um usuário específico.
    • PUT /api_tempo/user/update/:pk/: Atualiza os dados de um usuário existente.
    • DELETE /api_tempo/user/delete/:pk/: Exclui um usuário existente.
  • Previsão do Tempo:
    • GET /api_tempo/weather/: Retorna a lista de previsões do tempo.
    • POST /api_tempo/weather/create/: Cria uma nova previsão do tempo.
    • GET /api_tempo/weather/get/:pk/: Retorna os detalhes de uma previsão do tempo específica.
    • PUT /api_tempo/weather/update/:pk/: Atualiza os dados de uma previsão do tempo existente.
    • DELETE /api_tempo/weather/delete/:pk/: Exclui uma previsão do tempo existente.
    • GET /api_tempo/weather/city/:city/: Retorna previsões do tempo para uma cidade específica.

Funcionalidades de Autenticação

As funcionalidades de login, logout e autenticação foram implementadas utilizando Django e Django Rest Framework. Aqui está uma breve explicação de como cada funcionalidade foi implementada:

  • Login: A funcionalidade de login foi implementada na página de login. Os usuários podem inserir seu email e senha para autenticar-se na API.
  • Logout: A funcionalidade de logout permite que os usuários façam logout de suas contas, limpa o token JWT do cookie e os redireciona de volta para a página de login.
  • Autenticação: A autenticação é realizada utilizando tokens JWT. Um middleware intercepta todas as requisições para verificar se o usuário está autenticado. Se estiver, o usuário é definido no objeto de requisição para uso posterior.
