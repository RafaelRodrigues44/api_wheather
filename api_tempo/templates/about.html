This API, built with Django and Django Rest Framework (DRF), is a RESTful API that follows the MVT (Model-View-Template) pattern.

The MVT pattern is a variation of the MVC (Model-View-Controller) pattern, commonly used in web frameworks like Django. Here is a brief explanation of each component:

  • Model: Represents the data structure of the application. In the case of this API, the models define the structure of weather forecasts and users.
  • View: Handles the presentation logic and user interaction. Views are responsible for processing HTTP requests and returning appropriate responses. Views in this context are implemented as Django view classes.
  • Template: Provides the presentation structure for view responses. Templates are HTML files that can include dynamic data rendered by the views.

Additionally, this API uses the NoSQL database MongoDB to store weather forecast data. MongoDB is a common choice for applications that require scalability and flexibility in data schema.

Available Endpoints

  • Users:
    • GET /api_tempo/users/: Returns the list of users.
    • POST /api_tempo/user/create/: Creates a new user.
    • GET /api_tempo/user/get/:email/: Returns the details of a specific user.
    • PUT /api_tempo/user/update/:pk/: Updates an existing user's data.
    • DELETE /api_tempo/user/delete/:pk/: Deletes an existing user.
  • Weather Forecast:
    • GET /api_tempo/weather/: Returns the list of weather forecasts.
    • POST /api_tempo/weather/create/: Creates a new weather forecast.
    • GET /api_tempo/weather/get/:pk/: Returns the details of a specific weather forecast.
    • PUT /api_tempo/weather/update/:pk/: Updates an existing weather forecast.
    • DELETE /api_tempo/weather/delete/:pk/: Deletes an existing weather forecast.
    • GET /api_tempo/weather/city/:city/: Returns weather forecasts for a specific city.

Authentication Features

The login, logout, and authentication features were implemented using Django and Django Rest Framework. Here is a brief explanation of how each feature was implemented:

  • Login: The login functionality was implemented on the login page. Users can enter their email and password to authenticate with the API.
  • Logout: The logout functionality allows users to log out of their accounts, clears the JWT token from the cookie, and redirects them back to the login page.
  • Authentication: Authentication is done using JWT tokens. Middleware intercepts all requests to check if the user is authenticated. If authenticated, the user is set in the request object for later use.
